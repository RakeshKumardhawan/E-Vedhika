# E-VEDHIKA Agent Instructions

1. **What's New Updates (Nightly Approval)**: Do NOT add new versions to `SYSTEM_UPDATES` for every small change. All updates for a single day must be grouped into a single Version number (e.g., v1.5.2). Present the proposed text to the user for approval at the end of the day (or when requested) before adding it to `src/App.tsx`. Never create multiple 'V' entries for the same day.

3. Make sure to adhere to existing Tailwind design patterns and responsive breakpoints.
